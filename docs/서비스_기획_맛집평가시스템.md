# 회사 맛집 평가 시스템 - 서비스 기획서 (심화)

## 1. 사용자 이해: "매일 12시, 우리의 작은 고민"

### 1.1 점심시간 의사결정의 숨겨진 복잡성

#### 표면적 고민
- **제한된 시간**: 1시간 (이동 10분 + 대기 5분 + 식사 30분 + 복귀 10분)
- **선택 과부하**: 회사 반경 500m 내 평균 50개 식당
- **정보 신뢰도**: 네이버/카카오 리뷰는 일반인 기준, 우리와 다름

#### 심층적 니즈
- **심리적 안정감**: "오늘도 실패하면 오후가 우울해"
- **사회적 압박**: "나 때문에 다들 맛없는 곳 갔다고 할까봐"
- **경제적 부담**: "월말이라 비싼 곳은 부담스러워"
- **건강 고민**: "매일 기름진 음식만... 하지만 샐러드는 아쉬워"
- **네트워킹**: "다른 팀 사람들과도 친해지고 싶은데"

### 1.2 실제 점심시간 행동 패턴 분석

#### 데이터 기반 인사이트 (가상 설문 500명)
- **11:30-11:45**: 35%가 메뉴 고민 시작 ("슬슬 뭐 먹지?")
- **11:45-11:55**: 45%가 동료와 상의 ("우리 팀 뭐 먹을래?")
- **11:55-12:05**: 최종 결정 및 이동 ("그냥 평소 가던 곳")
- **결과**: 65%가 같은 5곳 내에서 반복

### 1.3 핵심 사용자 페르소나 (심화)

#### 👤 "효율적인 김대리" (30대 초반, 개발팀)
- **니즈**: 빠르게 먹고 돌아오고 싶음
- **관심사**: 웨이팅 시간, 서빙 속도, 혼밥 가능 여부
- **행동패턴**: 같은 곳 반복 방문, 안전한 선택 선호
- **불만**: "새로운 곳 가보고 싶은데 실패가 두려워요"

#### 👤 "맛집 탐험가 이과장" (30대 후반, 마케팅팀)
- **니즈**: 새로운 맛집 발견의 즐거움
- **관심사**: 맛, 분위기, 인스타그램 가능한 비주얼
- **행동패턴**: 적극적인 리뷰 작성, SNS 공유
- **불만**: "동료들이 제 추천을 신뢰하지 않아요"

#### 👤 "신입사원 박사원" (20대 후반)
- **니즈**: 선배들과 어울리고 싶음
- **관심사**: 가성비, 팀 분위기, 회식 장소
- **행동패턴**: 선배 따라가기, 리뷰 참고 의존
- **불만**: "어디가 좋은지 정말 모르겠어요"

#### 👤 "점심약속 많은 김부장" (40대)
- **니즈**: 손님 접대, 팀 회식에 적합한 장소
- **관심사**: 룸 여부, 주차, 단체석, 예약 가능
- **행동패턴**: 검증된 곳 선호, 실패 리스크 최소화
- **불만**: "중요한 미팅인데 식당이 별로면 곤란해요"

#### 👤 "점심 난민 김개발" (28세, 주니어 개발자)
- **실제 행동**: 매일 11:50까지 코딩 → 급하게 나가서 줄 긴 곳 포기
- **숨은 니즈**: "미리 웨이팅 정보만 알아도..."
- **제약사항**: 스탠드업 미팅 12:30 복귀 필수
- **실패 경험**: "어제도 컵라면... 진짜 밥 먹고 싶다"

#### 👤 "점심 인플루언서 박마케터" (33세, 마케팅 팀장)
- **실제 행동**: 인스타용 맛집 탐방 → 팀원들에게 공유
- **숨은 니즈**: "내 리뷰를 회사 사람들이 믿고 따라왔으면"
- **영향력**: 평균 3-4명과 함께 이동
- **보상 기대**: "맛집 큐레이터로 인정받고 싶어"

#### 👤 "혼밥러 이디자인" (30세, UI/UX 디자이너)
- **실제 행동**: 혼자 조용히 먹을 곳 찾기
- **숨은 니즈**: "혼밥 눈치 안 보는 곳 정보"
- **선호**: 바 테이블, 1인 메뉴, 태블릿 주문
- **회피**: "단체석만 있는 곳은 부담스러워"

## 2. 문제 정의: "왜 점심 결정이 어려운가?"

### 2.1 정보의 비대칭성

#### 외부 리뷰 서비스의 한계
- **네이버/카카오**: 주말 방문객, 데이트 목적 리뷰가 대부분
- **배달앱**: 배달 음식 위주, 매장 식사와 품질 차이
- **인스타그램**: 비주얼 중심, 실용 정보 부족

#### 우리가 진짜 알고 싶은 정보
- "지금 가면 자리 있을까?"
- "주문하고 몇 분 걸릴까?"
- "1만원으로 배부를까?"
- "혼자 가도 어색하지 않을까?"
- "우리 팀장님 입맛에 맞을까?"

### 2.2 점심 메뉴 결정의 실제 과정 (심화)
```
[11:30] 트리거 발생
  └─ "슬슬 배고픈데..." (신체적)
  └─ "옆팀이 나가네" (사회적)
  └─ 캘린더 알림 (시스템적)

[11:35] 1차 필터링 - 제외 조건
  └─ "어제 먹은 건 싫어" (중복 회피)
  └─ "비 오는데 멀리는..." (날씨 제약)
  └─ "월말이라 비싼 건..." (예산 제약)

[11:40] 2차 탐색 - 동료 협의
  └─ 슬랙/카톡: "뭐 먹을래?"
  └─ 침묵 → "아무거나" → 다시 원점
  └─ 의견 충돌 → 다수결 or 포기

[11:45] 3차 검증 - 리스크 체크
  └─ "거기 브레이크타임 아니야?"
  └─ "지금 가면 줄 서겠지?"
  └─ "리뷰 보니까 별로던데?"

[11:50] 최종 결정 - 타협점
  └─ 안전한 선택 (65%): "그냥 늘 가던 곳"
  └─ 도전 (20%): "새로 생긴 곳 가볼까?"
  └─ 포기 (15%): "편의점/배달"
```

### 2.3 의사결정 영향 요인 (심화 분석)

#### 🕐 시간 제약형 사용자
1. **거리** (도보 5분 이내)
2. **웨이팅** (대기 없음)
3. **서빙 속도** (주문 후 10분 이내)
4. **가격** (1만원 이하)
5. **맛** (평균 이상이면 OK)

#### 🍽️ 경험 추구형 사용자
1. **맛** (4.0 이상 평점)
2. **새로움** (안 가본 곳)
3. **분위기** (인테리어, 음악)
4. **거리** (도보 10분까지 OK)
5. **가격** (1.5만원까지 OK)

#### 💰 가성비형 사용자
1. **가격** (8천원 이하)
2. **양** (배부르게)
3. **거리** (가까울수록 좋음)
4. **맛** (가격 대비 만족)
5. **웨이팅** (5분 이내)

#### 외부 요인
- **날씨**: 비/눈 → 가까운 곳 선호 (+200m 감소)
- **요일**: 월요일(든든) vs 금요일(특별) vs 수요일(아무거나)
- **계절**: 여름(시원한) vs 겨울(따뜻한)
- **회사 일정**: 회의 전(가볍게) vs 야근 예정(든든하게)

#### 내부 요인
- **기분**: 스트레스(매운 음식) vs 피곤(간단한 음식)
- **건강**: 다이어트 중 vs 벌크업 중
- **경제**: 월초(여유) vs 월말(절약)
- **사회**: 혼자 vs 2-3명 vs 단체

## 3. 솔루션: "우리 회사만의 점심 비서"

### 3.1 핵심 가치: "Trust, Speed, Together"

#### Trust - 우리 회사 사람들의 진짜 데이터
**내부 평점/리뷰 시스템**
- 실명 or 부서명 리뷰 (신뢰도 ↑)
- 방문 인증: 점심시간 GPS + 결제 영수증
- 맥락 있는 평가: "2시 미팅 전 빠른 식사로 good"
- 동료 추천: "개발팀 선호도 85%"

**외부 데이터 스마트 통합**
- 네이버/카카오 기본 정보 (메뉴, 가격, 위치)
- 실시간 웨이팅 (네이버 예약 API 연동)
- 리뷰 신뢰도 필터링 (점심시간대 리뷰만)
- 크로스 체크: 내부 평점 vs 외부 평점 비교

#### Speed - 3초 추천, 1분 결정

**즉시 추천 모드**
- "지금 바로": 도보 5분, 대기 없음, 영업 중
- "오늘의 안전카드": 평점 4.0+, 10회+ 방문
- "1분 결정": 3개만 보여주기 (선택 피로 최소화)

**스마트 필터**
- 컨텍스트 인식: 날씨 + 시간 + 내 일정
- 이전 기록: "수요일엔 주로 한식"
- 그룹 선호: "우리 팀은 매운 걸 좋아해"

#### Together - 함께하는 점심 문화

**런치메이트 (점심 친구 찾기)**
- 익명 점심 제안: "오늘 12:15 일식 같이 드실 분"
- 관심사 매칭: "#맵부심 #혼밥_아닌_혼밥"
- 부서 믹스: "타 부서 네트워킹 런치"
- 심리적 안전장치:
  - 최소 2명 모이면 성사
  - 취소 가능 (11:50까지)
  - 평가/리뷰 없음 (부담 제거)

### 3.2 차별화된 기능 설계

#### Phase 1: Core MVP - "일단 시작하기"
**1. 기본 지도 + 식당 정보**
- 회사 중심 500m 네이버 지도
- 외부 API 기본 정보 (영업시간, 메뉴, 가격)
- 도보 시간 자동 계산
- 카테고리 필터 (한/중/일/양/기타)

**2. 내부 리뷰 시스템 v1**
- 5점 척도 + 100자 한 줄 평
- 사진 1장 업로드
- 익명/실명 선택
- "다시 갈래요" 버튼
- 리뷰 인센티브:
  - 월 5개 작성 시 커피 쿠폰
  - 베스트 리뷰어 뱃지
  - 리뷰 포인트 → 복지몰 전환

**3. 오늘의 추천 (단순 버전)**
- 날씨 기반: "비 오는 날 뜨끈한 국밥"
- 요일 기반: "금요일은 특별하게"
- 인기 순위: "이번 주 최다 방문 TOP 5"

**4. 빠른 결정 도구**
- "룰렛": 랜덤 추천 (재미 요소)
- "따라가기": 좋아하는 동료의 오늘 선택
- "안 가본 곳": 도전 뱃지 획득

#### Phase 2: Growth - "활성화하기"
**1. 런치메이트 시스템**
- 점심 모임 생성/참여
- 관심사 태그: #다이어트 #맵부심 #수다떨기
- 정기 모임: "매주 수요일 일식 모임"
- 부담 없는 만남:
  - 프로필 최소화 (부서/닉네임만)
  - 일회성 만남 OK
  - 평가 없음

**2. 스마트 개인화**
- AI 취향 학습: "금요일엔 주로 양식을"
- 건강 모드: 칼로리/영양 정보
- 예산 관리: "이번 달 점심 지출 트래킹"
- 알러지/종교/채식 필터

**3. 팀 & 그룹 기능**
- 팀 정기 회식 관리
- 투표 시스템 (슬랙/팀즈 연동)
- 부서별 맛집 지도
- 팀 평균 선호도 분석

**4. 리워드 시스템**
- 포인트 적립: 리뷰, 사진, 정보 제공
- 레벨 시스템: 맛집 탐험가 Lv.1~10
- 월간 챌린지: "이달의 신규 오픈 5곳 도전"
- 뱃지: 얼리버드(11:30 전), 개척자(처음 리뷰)

#### Phase 3: Expansion - "확장하기"

**1. 예측 & 자동화**
- 캘린더 연동: "2시 미팅 있으니 가벼운 메뉴 추천"
- 패턴 예측: "목요일 12시는 보통 김과장님과"
- 자동 예약: "매주 금요일 단골집 자동 예약"

**2. 소셜 강화**
- 맛집 피드: 동료들의 실시간 점심
- 먹스타그램: 사내 음식 사진 공유
- 맛집 토크: 식당별 실시간 채팅방

**3. 외부 서비스 연동**
- 결제 연동: 삼성/애플 페이
- 예약 연동: 네이버 예약, 캐치테이블
- 배달 연동: 회사로 배달 (야근용)

## 4. 사용자 시나리오 (심화)

### 시나리오 1: "처음 사용하는 신입사원 A씨"
```
[Day 1 - 온보딩]
- 회사 이메일로 자동 가입
- 팝업: "우리 회사 인기 맛집 TOP 10 보기"
- 부서(개발팀) 자동 인식 → "개발팀이 좋아하는 맛집"

[Day 3 - 첫 사용]
- 11:45 알림: "첫 점심 추천받아보세요! 🍽"
- 3초 설문: 매운거? 안매운거? → 한식 추천
- 선배 리뷰 확인: "신입 때 자주 갔어요. 착한 가격"
- 첫 방문 인증 → 웰컴 포인트 500P

[Day 10 - 정착]
- 런치메이트 추천: "같은 입사 동기 3명이 모였어요"
- 참여 → 동기들과 첫 점심 → 자동 그룹 생성
```

### 시나리오 2: "매일 점심 고민하는 김대리 (3년차)"
```
[월요일 11:30]
- 푸시 알림: "날씨 추워요. 뜨끈한 국물 어때요?"
- 맞춤 추천: 최근 일주일 면 요리 안 먹음 → 라멘 추천
- "따라가기": 존경하는 팀장님 오늘 선택 확인

[수요일 11:45 - 결정 장애]
- "룰렛 돌리기" → 랜덤 3곳 중 선택
- 선택 후 → "모험가 포인트 +10"

[금요일 12:00 - 런치메이트]
- 매칭 알림: "#수다떨기 좋아하는 마케팅팀 2명"
- 처음 만나는 타부서 동료들과 점심
- 종료 후: "즐거운 점심이었나요?" → 👍
- 다음 주 추천: "지난주 만난 분들과 정기 모임 만들기"
```

### 시나리오 3: "팀 회식 담당 이과장"
```
[회식 일주일 전]
```

## 5. 차별화 전략

### 5.1 "내부 데이터의 압도적 신뢰도"
**왜 직원 리뷰가 다른가?**
- 같은 시간 제약 (1시간)
- 같은 예산 수준 (회사원 점심값)
- 같은 위치 (회사 주변)
- 같은 목적 (빠르고 맛있게)

**신뢰도 강화 방안**
- 실명/부서명 공개 선택
- 방문 자동 인증 (GPS + 시간)
- 히스토리 공개 ("52번 방문한 단골")
- 전문성 표시 ("일식 전문가", "맵부심 레벨10")

### 5.2 "런치메이트: 혼자보다 함께"
**심리적 장벽 해결**
- 부담 없는 만남: "밥만 먹고 헤어져도 OK"
- 익명성 보장: 닉네임만 공개
- 소수 매칭: 2-4명 소규모
- 선택적 참여: 관심 있을 때만

**매칭 알고리즘**
- 관심사 기반: #수다 #조용히 #네트워킹
- 음식 취향: 비슷한 선호도
- 시간대: 11:30파 vs 12:00파 vs 12:30파
- 경험치: 신입끼리 or 선후배 믹스

### 5.3 "하이브리드 정보: 내부+외부"
**스마트 통합 전략**
```
기본 정보 (외부 API)
  ├── 네이버/카카오: 위치, 메뉴, 가격, 사진
  ├── 배달앱: 리얼 메뉴 사진, 가격
  └── 인스타: 최신 분위기, 신메뉴

실시간 정보 (크롤링/API)
  ├── 네이버 예약: 웨이팅 현황
  ├── 구글맵: 실시간 혼잡도
  └── 날씨 API: 날씨별 추천

검증된 정보 (내부 데이터)
  ├── 직원 평점: 신뢰도 높은 평가
  ├── 실제 경험: "점심시간 특화 리뷰"
  └── 부가 정보: "혼밥 OK", "카드 안됨" 등
```

## 8. 리스크 관리 (심화)

### 8.1 Critical Risks

**"초기 콘텐츠 부족"**
- 리스크: 리뷰 없어서 신뢰도 낮음
- 대응: 
  - 초기 시드 리뷰 (자원봉사 50명)
  - 외부 리뷰 임시 활용
  - 얼리어답터 인센티브 강화

**"개인정보 이슈"**
- 리스크: 실명 리뷰 거부감
- 대응:
  - 익명 옵션 제공
  - 부서명만 공개
  - GDPR 준수

**"런치메이트 실패"**
- 리스크: 어색한 만남 거부감
- 대응:
  - 최소 2명 매칭
  - 즉석 취소 가능
  - 부담 없는 UX

### 8.2 Operational Risks

**"데이터 정확성"**
- 잘못된 영업시간 → 실시간 검증
- 가격 변동 → 주기적 업데이트
- 폐업 정보 → 사용자 제보

**"기술 의존성"**
- 네이버 API 제한 → 캐싱, 다중 API
- Azure 장애 → 백업 인증
- 서버 과부하 → Auto-scaling
